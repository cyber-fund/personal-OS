# Create Project Skill

Scaffold a self-contained project directory under `./vault/private/projects/<slug>/`. Each project gets its own `CLAUDE.md` so any nested Claude invocation inside the folder understands what the project is for without needing to re-read the whole workspace.

**Usage:**
- `/cyber-create-project "Project Name"` — create with the given name
- `/cyber-create-project` — ask the user for a name first

---

## Workflow

### Phase 0: GET NAME

If no name was passed, ask the user for one. Accept natural-language input (e.g. "Acme Labs deal" or "Q2 content push").

### Phase 1: SLUGIFY

Convert the name to a URL-safe slug: lowercase, replace spaces/punctuation with `-`, collapse repeats, trim.

- "Acme Labs deal" → `acme-labs-deal`
- "Q2 content push" → `q2-content-push`

### Phase 2: CHECK FOR CONFLICT

If `./vault/private/projects/<slug>/` already exists, stop and report the path. Do not overwrite. Offer to open the existing project instead.

### Phase 3: CREATE STRUCTURE

Create:

```
./vault/private/projects/<slug>/
├── CLAUDE.md          # project-scoped operating manual
├── README.md          # overview
├── status.md          # current state, last updated
├── decisions.md       # rationale log
├── context/           # research, notes, references
└── deliverables/      # generated outputs for this project
```

Use the templates in `artifacts/` (`CLAUDE.md`, `README.md`, `status.md`, `decisions.md`). Substitute `{{name}}` and `{{slug}}` in the templates. Fill `{{date}}` with today's date (`YYYY-MM-DD`).

### Phase 4: LINK FROM GTD.md

Append one line to `./vault/GTD.md` under the `# Now` section:

```
- [Project: {{name}}](./vault/private/projects/{{slug}}/README.md)
```

If a `# Now` section doesn't exist, create it at the top.

### Phase 5: UPDATE PROJECTS INDEX

Add this project to `./vault/private/projects/INDEX.md` (create the file if missing). Use the project's `## Summary` from `status.md` as the entry body:

```
## {{name}} — active (updated {{date}})
[Summary text from status.md]
→ [README](./{{slug}}/README.md) · [status](./{{slug}}/status.md)
```

The full index is regenerated by `/cyber-context`; here you just add the new project so it appears immediately.

### Phase 6: REPORT

Output:

```
Project created: ./vault/private/projects/<slug>/

Structure:
  CLAUDE.md         project operating manual
  README.md         overview
  status.md         current state
  decisions.md      rationale log
  context/          research and notes
  deliverables/     outputs

Linked from: GTD.md (# Now), projects/INDEX.md

Next:
  - /cyber-research "<topic>" to seed context
  - /cyber-brief to see this project in the priority list
```

---

## Error Handling

| Error | Response |
|-------|----------|
| No name provided and user declines to give one | Exit without creating anything |
| `./vault/private/projects/` doesn't exist | Run `/cyber-setup` first, or create the directory inline and continue |
| Slug collides with an existing project | Report path, don't overwrite, suggest a different name |
| No write permissions | Report the path that failed and ask the user to check permissions |

---

## Notes

- Keep projects self-contained — all research, notes, and deliverables for the project live under its folder, not scattered across `./content/`.
- The top-level `./content/` folder is for one-off artifacts (daily briefs, ad-hoc research). Anything that spans sessions or has a lifecycle belongs in a project.
- The project-level `CLAUDE.md` points back to the root files (`GTD.md`, `cyber.md`, `who-am-i.md`) so Claude always has global context too.
