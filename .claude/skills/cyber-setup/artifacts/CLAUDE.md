# Vault — Operating Manual

This is the user's personal vault. Claude reads this file at the start of every task to understand **where things live** and **how to work here**.

## Where context is stored

Read these at the start of any relevant task so work reflects who the user is and what matters:

- `private/context/who-am-i.md` — identity, role, priorities, handles
- `private/context/cyber.md` — organization context
- `private/context/style/tone-of-voice.md` — writing-style profile
- `private/context/{telegram,emails,calls}/` — synced source data, per channel
- `content/inbox/` — daily raw deltas from all sources (most recent first)

## Where projects are stored

Multi-session work lives under `private/projects/<slug>/`. Scaffold one with `/cyber-create-project`. Each project folder contains:

- `CLAUDE.md` — project-scoped instructions (read when working inside the project)
- `README.md` — what it is, why it exists, success criteria
- `status.md` — current state + a rolling `## Summary` (what shipped, what's next, why)
- `decisions.md` — append-only rationale log (date · context · decision · why)
- `context/` — research and notes
- `deliverables/` — generated outputs

`private/projects/INDEX.md` is a rollup of every project's `## Summary`, regenerated by `/cyber-context`. It's the place to look for insights across all project work — but the source of truth is each project's `status.md`.

## Where generated artifacts go

- `content/briefs/` — daily situational briefs
- `content/research/` — research reports
- `content/drafts/` — draft posts
- `content/summaries/` — project-work summaries / insight digests
- `GTD.md` — living to-do list (read before tasks; move items between sections, don't delete)

## Rules

- All personal data stays in `private/` — nothing shared or uploaded without explicit permission.
- When writing on the user's behalf, apply the tone-of-voice profile.
- **Keep project state current**: update `status.md` (especially its `## Summary`) as work moves, and append to `decisions.md` whenever a real decision is made. This running record is what feeds `INDEX.md` and makes project work summarizable for insights.
- Keep `GTD.md` updated.

## Skills

Run skills with `/skill-name`. They live in the repo's `.claude/skills/`. Use `/cyber-context` for a live cross-project dashboard.
